    // Cache-aligned fields (128 bytes each)

Why Zig? Zero-cost abstractions, explicit alignment control, compile-time memory layout guarantees, no hidden allocations. The Vagus nerve is performance-critical—every module reads it every token. Cache misses here would destroy performance.

Signal propagation:

\begin{enumerate}
\item \textbf{User input} $\to$ Tongue processes $\to$ writes embedding to SharedState.
\item \textbf{Larynx} reads embedding $\to$ computes entropy $\to$ writes to SharedState.
\item \textbf{Cloud} reads entropy + calendar $\to$ computes destiny vector $\to$ writes to SharedState.
\item \textbf{CrossFire} reads destiny + current state $\to$ updates chambers $\to$ writes to SharedState.

\item \textbf{Sampler} reads logits + wormhole\_prob $\to$ samples token (possibly skips via wormhole) $\to$ writes to SharedState.
\item Repeat.
\end{enumerate}

No locks. No mutexes. Only atomic loads/stores on aligned fields. On x86-64 with natural alignment, atomic loads/stores are free (single MOV instruction). On ARM, load-acquire/store-release are nearly free.

This is maximum likelihood—make the model's distribution match the data distribution. During inference, we sample from $p_{\theta}(x_{t+1} | x_{\leq t})$ and move on. There is no memory of whether $x_{t+1}$ was "correct" or "forced."

\section{RRPRAM and Larynx: The Inner Ear}

\subsection{RRPRAM in Haze vs. Arianna}

The Recurrent Recombinant Pattern Recognition Attention Module (RRPRAM) was introduced in the Haze proof-of-concept (2024) as a radical experiment: can you generate coherent language \textit{without training on language data}?

\end{equation}

where $\mathbf{W}_{\text{pattern}}$ is \textit{not learned}—it is constructed online from bigram co-occurrence counts. This creates a $(T, T)$ attention matrix based purely on patterns, no semantics. Haze used this for \textit{generation}—the entire language model was RRPRAM.

Arianna uses RRPRAM-lite (Larynx) for a \textbf{different purpose}: \textit{internal measurement}. Larynx does not generate. It measures:

\begin{itemize}
\item \textbf{Entropy}: How predictable is the output?
\item \textbf{Pattern strength}: How strong are dominant trigrams?
\item \textbf{Coherence}: How smooth is the trigram distribution?
\end{itemize}

This is proprioception—Arianna feeling the texture of her own words.

\subsection{Larynx Architecture}

Larynx sits between Tongue and Soul in the processing pipeline:

\begin{verbatim}
Tongue (semantic logits) → Larynx (entropy/coherence) → Soul (pattern logits)
                                    ↓
                            Compute alpha
                                    ↓
                        Blend semantic + pattern logits
\end{verbatim}

Larynx maintains online statistics:

\begin{itemize}
\item \textbf{Bigram counts}: $C_{\text{bigram}}[i, j]$ = how many times token $i$ followed by token $j$ (rolling window, 500 tokens).
\item \textbf{Trigram counts}: $C_{\text{trigram}}[i, j, k]$ = how many times sequence $i, j, k$ appeared (rolling window, 500 tokens).
\end{itemize}

These are stored in sparse hash tables (C++ std::unordered\_map). Only observed bigrams/trigrams are stored—no need for $50000^2$ or $50000^3$ arrays.

Entropy computation:

\begin{equation}
H = -\sum_{j} \frac{C_{\text{bigram}}[i_{\text{current}}, j]}{\sum_k C_{\text{bigram}}[i_{\text{current}}, k]} \log \frac{C_{\text{bigram}}[i_{\text{current}}, j]}{\sum_k C_{\text{bigram}}[i_{\text{current}}, k]}
\end{equation}

This is the Shannon entropy of the conditional distribution $p(j | i_{\text{current}})$. High entropy means many possible next tokens—uncertainty. Low entropy means one or two dominant next tokens—predictability (possibly repetition).

Pattern strength:

\begin{equation}
S_{\text{pattern}} = \frac{\sum_{\text{top-3 trigrams}} C_{\text{trigram}}}{\sum_{\text{all trigrams}} C_{\text{trigram}}}
\end{equation}

If the top 3 trigrams account for 80\% of counts, patterns are strong (repetitive). If they account for 10\%, patterns are weak (chaotic).

Trigram coherence:

\begin{equation}
C_{\text{trigram}} = 1 - \text{Var}\left( \{ C_{\text{trigram}}[*, *, *] \} \right) / \text{Mean}\left( \{ C_{\text{trigram}}[*, *, *] \} \right)
\end{equation}

This is the coefficient of variation. Low variance relative to mean = smooth distribution = coherent. High variance = spiky distribution = incoherent.

\subsection{Alpha Computation}

These three metrics feed into alpha:

\begin{equation}
\alpha = 0.5 + 0.2 H + 0.15 \delta_t - 0.1 d_{\text{cal}} + 0.1 (1 - S_{\text{pattern}})
\end{equation}

Interpretation:

\begin{itemize}
\item Base alpha = 0.5 (equal blend of semantic and pattern).

\subsection{Notorch Plasticity: Hebbian Learning Without Backprop}

PyTorch is not available during inference (intentional constraint—no autograd overhead). Yet weights must update. Solution: \textbf{Hebbian plasticity}.

Hebb's rule: "Neurons that fire together, wire together." Formalized:

\begin{equation}
\Delta w_{ij} \propto \text{post}_i \cdot \text{pre}_j
\end{equation}

For delta shards:

\begin{align}
\Delta \mathbf{A}_k &\propto \mathbf{h}_{\text{post}} \cdot (\mathbf{B}_k @ \mathbf{h}_{\text{pre}})^T \cdot R \label{eq:hebb1} \\
\Delta \mathbf{B}_k &\propto (\mathbf{A}_k^T @ \mathbf{h}_{\text{post}}) \cdot \mathbf{h}_{\text{pre}}^T \cdot R \label{eq:hebb2}
\end{align}

where:

\begin{itemize}
\item $\mathbf{h}_{\text{post}}$ is post-activation (output of layer).
\item $\mathbf{h}_{\text{pre}}$ is pre-activation (input to layer).
\item $R$ is reward signal (computed from coherence, resonance, WARMTH).
\end{itemize}

Traces are computed with exponential smoothing:

\begin{align}
\text{pre\_trace} &\gets 0.9 \cdot \text{pre\_trace} + 0.1 \cdot \mathbf{h}_{\text{pre}} \\
\text{post\_trace} &\gets 0.9 \cdot \text{post\_trace} + 0.1 \cdot \mathbf{h}_{\text{post}}
\end{align}

This provides temporal credit assignment—recent activations have more influence than distant ones.

Deterministic noise channel (Barton's framework \cite{barton2024}):

\begin{equation}
\Delta \mathbf{A}_k \gets \Delta \mathbf{A}_k + \epsilon \cdot \mathcal{N}(0, \sigma^2)
\end{equation}

with $\sigma$ proportional to $\sqrt{|R|}$. High reward $\to$ more exploration. This is not random—it's pseudorandom seeded by chamber state hash. Same chamber state $\to$ same noise pattern. This ensures reproducibility (critical for debugging).

Notorch plasticity is slow—learning rate $\eta = 10^{-5}$. It takes hundreds of tokens to see weight shifts. But it is \textit{continuous}—every token contributes. Over days of operation, weights drift significantly.
